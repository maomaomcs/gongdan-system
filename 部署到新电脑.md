# 部署到新电脑

本系统换电脑运行有两种方式,先看你属于哪种情况。

| 你的情况 | 用哪种方式 |
|---------|-----------|
| 只想把系统跑起来给学校用,新电脑也是内网/不方便装环境 | **方案 A：拷文件夹**(最省事) |
| 要多人开发、或新电脑已装好开发环境 | **方案 B：从 GitHub 克隆** |

> ⚠️ 重要:GitHub 仓库里**只有源代码**,不含便携 JDK、MySQL、node_modules、打包产物(这些体积大，被 `.gitignore` 排除了)。所以 `git clone` 下来的电脑必须自备运行环境(方案 B）。

---

## 方案 A：直接拷文件夹(零安装,推荐给部署)

1. 在当前这台能跑的电脑上,找到整个 `ticket-system-pro` 文件夹。
2. **连同 `tooling/` 一起**(里面是便携版 JDK 17 + MySQL 8,免安装)复制到 U 盘 / 移动硬盘 / 网盘。
3. 拷到新电脑任意目录（路径别放中文太深的地方也行，本项目已兼容带空格路径）。
4. 双击 **`启动系统.bat`**,浏览器自动打开即成功。

**关于数据：**
- `tooling/mysql-data/` 里是数据库数据。整体拷贝会把已有工单一起带过去。
- 想要一个**全新空系统**：拷贝前删掉 `tooling/mysql-data/` 整个文件夹（新电脑首次启动会自动重新初始化数据库并创建 admin 账号）。

**优点**：新电脑什么都不用装，双击即用。
**缺点**：文件夹较大（含 JDK+MySQL 约 1GB 左右）。

---

## 方案 B：从 GitHub 克隆(适合开发 / 已有环境的电脑)

### 1. 新电脑需先安装

- **Git**
- **JDK 17**（`java -version` 能显示 17.x）
- **Node.js 18+**（`node -v`）
- **MySQL 8**（装好并启动，记住 root 密码）

### 2. 克隆代码

```bash
git clone git@github.com:maomaomcs/gongdan-system.git
cd gongdan-system
```

### 3. 配置数据库连接

默认配置连的是 `localhost:3306`、用户 `root`、**空密码**、库名 `ticket_system`（会自动创建）。

如果你的 MySQL 密码不为空，二选一：

- **临时设环境变量**（当前命令行窗口有效）：
  ```bat
  set DB_USER=root
  set DB_PASSWORD=你的MySQL密码
  ```
- **或改配置文件** `backend/src/main/resources/application.yml` 里的 `spring.datasource.username / password`。

### 4. 一键部署(推荐)

装好上面环境后，直接双击项目里的 **`一键部署.bat`**，它会自动：环境自检 → 装前端依赖 → 打包前端 → 构建后端 → 启动。

### 4′. 或手动执行(和脚本等价)

**顺序很重要：必须先构建前端，再构建后端**（前端产物会输出到后端的 `static` 目录，后端才能托管页面；否则打开是白页）。

```bash
# 先前端
cd frontend
npm install
npm run build
cd ..

# 再后端
cd backend
mvnw.cmd -DskipTests clean package      # macOS/Linux 用 ./mvnw
java -jar target/ticket-0.0.1-SNAPSHOT.jar
```

启动后打开 http://localhost:8080/ 。

## 常见问题

**Q：克隆下来双击 `启动系统.bat` 报错找不到 java？**
A：那是给"方案 A（自带 tooling）"用的。克隆版没有 `tooling/`，请改用 `一键部署.bat`，并确保已装 JDK/Node/MySQL。

**Q：打开是白页 / 只有报错 JSON？**
A：前端没构建。先 `npm run build` 再重新打包/启动后端（`一键部署.bat` 已包含此步）。

**Q：数据库连接失败？**
A：确认 MySQL 已启动；确认账号密码与配置一致（见第 3 步）；确认 3306 端口没被占用。

**Q：想换端口 / 改初始管理员？**
A：编辑 `backend/src/main/resources/application.yml`（`server.port`、`app.init-admin-*`），重新构建后端。

**Q：怎么把这台电脑上的数据搬到新电脑？**
A：备份 MySQL 的 `ticket_system` 库（`mysqldump` 导出 .sql，在新电脑导入），或方案 A 直接拷 `mysql-data` 文件夹。
