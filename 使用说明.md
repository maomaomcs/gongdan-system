## 一、系统能做什么

| 角色 | 能做的事 |
|------|---------|
| **老师** | 手机扫码填报修单；凭工单号查处理进度 |
| **你（管理员）** | 看所有工单、按条件筛选、改状态、写处理记录、看统计 |

三种状态流转：**待处理 → 处理中 → 已解决**（也可关闭）。每个工单有唯一编号，如 `BX20260720-A1B2`。

---

## 二、第一次启动

进入文件夹 `ticket-system-pro`，**双击 `启动系统.bat`**。

会弹出两个黑色窗口（一个是数据库，一个是系统后端），**不要关它们**。几秒后浏览器会自动打开。

看到报修页面就成功了。两个地址：

- **报修入口（发给老师）**：`http://localhost:8080/`
- **管理后台（你自己用）**：`http://localhost:8080/login`　初始账号 **admin** / 密码 **admin123**

> 💡 首次登录后请立刻在「账号管理 → 修改我的密码」里改掉默认密码，并可新建属于自己的账号。

---

## 三、老师怎么报修

1. 手机扫二维码（或打开报修网址），进入报修页。
2. 填写：
   - **报修人姓名**（必填）
   - **联系方式**（选填，方便你联系）
   - **位置 / 教室**（必填，如"教学楼3楼302"）
   - **故障类型**（必填，下拉选：投影仪 / 电脑 / 网络 …）
   - **问题简述**（必填，一句话）
   - **详细描述**（选填）
   - **紧急程度**（普通 / 紧急）
3. 点"提交报修"，会显示一个**工单号**，让老师记下来。
4. 之后老师可以点"查询进度"，输入工单号看处理到哪一步了。

---

## 四、管理员怎么处理工单

1. 打开 `http://localhost:8080/login`，输入密码登录。
2. 进入**工单管理**页，能看到所有工单列表：
   - 待处理的、紧急的排在最前面。
   - 上方可按**状态 / 类型 / 紧急度**筛选，或输入关键词搜索（标题、位置、报修人、工单号都能搜）。
3. **点某一行**，右侧弹出详情面板：
   - 改 **状态**（待处理 → 处理中 → 已解决）
   - 填 **处理人**（谁在修）
   - 填 **解决方案 / 备注**
   - 点"保存处理结果"
4. 处理过程中可以随时**加跟进记录**（如"已联系厂商，明天上门"），形成完整的处理轨迹。

> 状态改成"已解决"时，系统会自动记录解决时间，用于统计平均处理时长。

## 五、查看数据统计

登录后点左侧**数据统计**，能看到：

- **三个概览卡**：工单总数、待处理/处理中数量、平均解决时长（小时）。
- **按状态分布**（饼图）
- **按故障类型**（哪类问题最多，柱状图）
- **按位置/教室**（哪里最坏，Top 15）
- **按月份趋势**（每月报修量）

> 这几张图就是你年底汇报工作量、申请设备更换预算的直接依据。

## 六、让老师扫码报修（重要）

系统跑在你这台电脑上，老师的手机要能访问，需在**同一个局域网/校园网**内。

**步骤：**

1. 在这台电脑上，按 `Win + R`，输入 `cmd` 回车，敲 `ipconfig` 回车。
2. 找到 **IPv4 地址**，例如 `192.168.1.50`。
3. 那么老师的报修网址就是：**`http://192.168.1.50:8080/`**
4. 用任意"网址转二维码"工具（微信、草料二维码等）把这个网址做成二维码。
5. 打印二维码，贴在各教室门口 / 讲台。老师扫码即可报修。

> ⚠️ 换了网络或电脑重启后 IP 可能变化。如需固定，建议给这台电脑设**静态 IP**（找网管配置）。

## 七、常用设置（改密码 / 故障类型 / 端口）

配置文件：`backend/src/main/resources/application.yml`，用记事本即可打开编辑。

**改管理密码**（强烈建议）：登录后台后，进「账号管理 → 修改我的密码」直接改，不用动配置文件。
也可在「账号管理 → 新增账号」为每位管理员建独立账号（用户名+密码），处理工单时能区分是谁。

`application.yml` 里的 `init-admin-username / init-admin-password` 只是**首次启动**创建初始账号用的，账号建好后改它无效（改密请用后台界面）。

**改故障类型下拉选项**：
```yaml
app:
  categories:
    - 电脑/主机
    - 投影仪
    - 你要加的类型
```

**改端口**（默认 8080，一般不用改）：
```yaml
server:
  port: 8080
```

改完 `application.yml` 后：**关闭"工单系统后端"窗口，重新双击 `启动系统.bat`** 即可生效（数据库不用重启）。

> 如果改了前端或后端**代码**（不是配置），需要先双击 `重新构建.bat` 打包，再启动。

## 八、数据备份

所有数据都存在 MySQL 里，两种备份方式任选：

**方式 A（最简单）**：先关闭"MySQL数据库"窗口，然后把整个 `tooling/mysql-data/` 文件夹复制到 U 盘/网盘。恢复时复制回去即可。

**方式 B（导出 SQL）**：命令行进入 `tooling/mysql-8.0.29-winx64/bin/`，运行：
```
mysqldump.exe -u root ticket_system > D:\备份_20260720.sql
```

建议每月备份一次。

## 九、关闭与重启

- **临时关闭**：关掉"MySQL数据库"和"工单系统后端"两个黑窗口即可。数据不会丢。
- **重新启动**：再次双击 `启动系统.bat`。
- **开机自动运行**：如需电脑一开机就自动跑，可把 `启动系统.bat` 放进"启动"文件夹（`Win+R` 输入 `shell:startup` 打开）。更稳妥的做法是做成 Windows 服务，需要时我可以帮你配。

## 十、常见问题 FAQ

**Q：浏览器打开是空白 / 打不开？**
A：确认那两个黑窗口还开着；等 10 秒后端才完全启动；刷新页面。仍不行就重启 `启动系统.bat`。

**Q：老师扫码打不开？**
A：① 手机和电脑要在同一网络；② 确认用的是电脑的局域网 IP（不是 localhost）；③ 电脑防火墙可能拦了 8080 端口，首次会弹窗，选"允许访问"。

**Q：忘了管理密码？**
A：打开 `application.yml` 看/改 `admin-password`，重启后端。

**Q：想加一个"处理人"账号分工？**
A：当前是单一管理员密码。如果需要多人账号、权限区分，告诉开发即可扩展。

**Q：数据会丢吗？**
A：只要不删 `tooling/mysql-data/` 文件夹就不会丢。定期备份更保险。

**Q：这套东西装了会不会拖慢电脑 / 弄乱系统？**
A：不会。Java 和 MySQL 都是"绿色版"放在 `tooling/` 文件夹里，没往系统里安装任何东西。不想要了，直接删掉整个 `ticket-system-pro` 文件夹即可，干干净净。

## 技术信息（给懂技术的人看）

- 前端：Vue 3 + Element Plus + ECharts，打包后由后端托管
- 后端：Spring Boot 3.4 + Spring Data JPA
- 数据库：MySQL 8.0（本地 3306）
- 运行：便携 JDK 17，单服务端口 8080
- 接口文档见 `README.md`
