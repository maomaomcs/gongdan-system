# 部署到 Linux 公网服务器

以部署到 **43.136.56.131:8082** 为例(腾讯云等 Linux 服务器通用)。

> ⚠️ **公网安全提醒(务必先看)**
> 放到公网 = 任何人都能访问,和只在校内局域网用不一样。部署前/后请务必做到:
> 1. **改掉管理员默认密码**(别用 admin/admin123);
> 2. **给 MySQL 设密码**,并只监听本机(不要对公网开放 3306);
> 3. 安全组**只放行 8082**(和你要用的 22/80/443),别把数据库端口暴露出去;
> 4. 有条件上 **HTTPS**(见文末 Nginx 方案),否则密码是明文传输;
> 5. 建议尽快处理"工单号防枚举"(公开查询接口可被遍历读到他人报修信息)——需要的话让我加。

## 一、服务器需要装什么

```bash
# JDK 17
sudo apt update && sudo apt install -y openjdk-17-jre-headless
# MySQL 8
sudo apt install -y mysql-server
sudo systemctl enable --now mysql
```

（CentOS/RockyLinux 用 `dnf install java-17-openjdk mysql-server`）

## 二、建数据库和账号(设密码,别用空密码)

```bash
sudo mysql
```
```sql
CREATE DATABASE ticket_system DEFAULT CHARACTER SET utf8mb4;
CREATE USER 'ticket'@'localhost' IDENTIFIED BY '你的强密码';
GRANT ALL PRIVILEGES ON ticket_system.* TO 'ticket'@'localhost';
FLUSH PRIVILEGES;
EXIT;
```

## 三、把程序传到服务器

系统是单个 jar(前端已打包进去),只需传一个文件。

**方式 A：从你 Windows 电脑上传**(jar 在 `backend/target/ticket-0.0.1-SNAPSHOT.jar`)
```bash
# 在 Windows 的项目目录执行(scp 随 git 自带)
scp backend/target/ticket-0.0.1-SNAPSHOT.jar root@43.136.56.131:/opt/ticket-system/app.jar
```

**方式 B：在服务器上从 GitHub 拉源码自行构建**(服务器需再装 git、nodejs、maven)
```bash
git clone git@github.com:maomaomcs/gongdan-system.git
cd gongdan-system/frontend && npm install && npm run build && cd ..
cd backend && ./mvnw -DskipTests clean package
cp target/ticket-0.0.1-SNAPSHOT.jar /opt/ticket-system/app.jar
```

准备目录:
```bash
sudo mkdir -p /opt/ticket-system/data/uploads
sudo useradd -r -s /sbin/nologin ticket 2>/dev/null || true
sudo chown -R ticket:ticket /opt/ticket-system
```

## 四、用 systemd 托管(开机自启、崩溃自动重启)

把仓库里 `deploy/ticket-system.service` 传到服务器,**改好里面的密码/路径**,然后:

```bash
sudo cp ticket-system.service /etc/systemd/system/
sudo nano /etc/systemd/system/ticket-system.service   # 改 DB_PASSWORD、ADMIN_INIT_PASSWORD
sudo systemctl daemon-reload
sudo systemctl enable --now ticket-system
sudo systemctl status ticket-system        # 看是否 running
sudo journalctl -u ticket-system -f        # 看实时日志
```

## 五、放行端口

1. **云平台安全组**:在腾讯云控制台给这台服务器的安全组加一条入站规则:允许 TCP **8082**(来源按需,内部用可限公司 IP)。
2. **服务器防火墙**(如启用了 ufw/firewalld):
   ```bash
   sudo ufw allow 8082/tcp        # ufw
   # 或 firewalld:
   sudo firewall-cmd --permanent --add-port=8082/tcp && sudo firewall-cmd --reload
   ```

## 六、访问

- 报修入口:http://43.136.56.131:8082/
- 管理后台:http://43.136.56.131:8082/login （用你在 `ADMIN_INIT_PASSWORD` 设的密码,首次登录后再到「账号管理」改)

## 九、(推荐)加 HTTPS + 域名

裸 IP + HTTP 密码是明文。若有域名,可用 Nginx 反代 + Let's Encrypt:

```nginx
server {
    listen 80;
    server_name your.domain.com;
    location / {
        proxy_pass http://127.0.0.1:8082;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        client_max_body_size 30m;   # 允许上传图片
    }
}
```
再用 `certbot --nginx` 申请证书即可。后端已开启 `forward-headers-strategy`,反代后能正确识别。
