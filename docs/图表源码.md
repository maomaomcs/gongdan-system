# 论文配图源码（PlantUML）

> 用法：
> 1. 在线渲染：打开 https://www.plantuml.com/plantuml ，把 `@startuml ... @enduml` 之间（含首尾）的内容粘贴进去，即可生成 PNG/SVG。
> 2. VS Code：安装 "PlantUML" 插件，`Alt+D` 预览，右键可导出图片。
> 3. 需要本地渲染时需装 Java + Graphviz。
>
> 三张核心图：系统架构图、数据库 E-R 图、用例图。另附报修流程状态图。

---

## 图1 系统总体架构图

```plantuml
@startuml
skinparam componentStyle rectangle
skinparam rectangle {
  BackgroundColor #FDF6E8
  BorderColor #A4232A
}
skinparam ArrowColor #7A1519

title 校园报障工单与IT资产管理系统 · 总体架构

actor "教师" as teacher
actor "管理员" as admin
actor "游客" as guest

package "表现层（Vue3 + Element Plus + PWA）" {
  [教师端\n报修 / 我的报修] as feUser
  [管理端\n工单 / 资产 / 统计 / 设置] as feAdmin
}

package "应用层（Spring Boot）" {
  [Controller 层\nREST 接口] as ctrl
  [拦截器\n鉴权 / 令牌] as intc
  [Service 层\n业务逻辑 / 事务] as svc
  [Repository 层\nSpring Data JPA] as repo
  [钉钉推送 / Excel 导入导出\n图片压缩 / 定时任务] as util
}

database "MySQL" as db
folder "本地文件\n上传图片" as fs
cloud "钉钉群机器人" as ding

teacher --> feUser
admin --> feAdmin
guest --> feUser

feUser --> ctrl : HTTP+JSON\n(Bearer Token)
feAdmin --> ctrl : HTTP+JSON\n(Bearer Token)
ctrl --> intc
intc --> svc
svc --> repo
svc --> util
repo --> db
util --> fs
util --> ding

note right of feAdmin
  前端打包后的静态资源
  由后端一并托管，
  生产仅运行一个 Jar(端口8082)
end note
@enduml
```

## 图2 数据库 E-R 图

```plantuml
@startuml
skinparam linetype ortho
hide circle
skinparam class {
  BackgroundColor #FDF6E8
  BorderColor #A4232A
}

entity "app_user 教师" as app_user {
  * id : bigint <<PK>>
  --
  username : varchar(64) <<U>>
  password_hash : varchar
  display_name : varchar
  phone : varchar
  enabled : boolean
}

entity "admin_user 管理员" as admin_user {
  * id : bigint <<PK>>
  --
  username : varchar(64) <<U>>
  password_hash : varchar
  enabled : boolean
}

entity "ticket 工单" as ticket {
  * id : bigint <<PK>>
  --
  code : varchar(32) <<U>>
  reporter : varchar
  location : varchar
  asset_no : varchar(64) <<FK弱>>
  category : varchar
  title : varchar
  status : varchar
  handler : varchar
  urgency : varchar
  user_id : bigint <<FK>>
  urge_count : int
  user_unread : boolean
  created_at : datetime
  resolved_at : datetime
}

entity "ticket_log 跟进" as ticket_log {
  * id : bigint <<PK>>
  --
  ticket_id : bigint <<FK>>
  content : varchar
  author : varchar
  created_at : datetime
}

entity "asset 资产" as asset {
  * id : bigint <<PK>>
  --
  asset_no : varchar(64) <<U>>
  type : varchar
  brand_model : varchar
  serial_no : varchar
  ip : varchar
  location : varchar
  owner : varchar
  purchase_date : date
  warranty_end : date
  status : varchar
}

entity "invite_code 邀请码" as invite_code {
  * id : bigint <<PK>>
  --
  code : varchar(32) <<U>>
  max_uses : int
  used_count : int
  enabled : boolean
}

entity "auth_token 登录令牌" as auth_token {
  * token : varchar <<PK>>
  --
  username : varchar
  role : varchar
  expires_at : datetime
}

app_user ||--o{ ticket : "提交"
ticket ||--o{ ticket_log : "包含"
asset ||--o{ ticket : "关联(asset_no)"

@enduml
```

> 说明：`asset` 与 `ticket` 通过资产编号 `asset_no` 弱关联（非外键约束，选填），体现"设备—工单"打通。`system_setting`（键值表）为独立配置表，图中省略。

## 图3 用例图

```plantuml
@startuml
left to right direction
skinparam packageStyle rectangle
skinparam usecaseBackgroundColor #FDF6E8
skinparam usecaseBorderColor #A4232A

actor 教师 as t
actor 管理员 as a
actor 游客 as g

rectangle 校园报障工单与IT资产管理系统 {
  usecase "注册/登录" as u1
  usecase "提交报修" as u2
  usecase "查看我的报修" as u3
  usecase "催单" as u4
  usecase "取消报修" as u5
  usecase "按工单号查询" as u6

  usecase "工单处理与跟进" as a1
  usecase "工单统计/导出" as a2
  usecase "资产台账管理" as a3
  usecase "资产批量导入/导出" as a4
  usecase "查看设备报障历史" as a5
  usecase "账号与邀请码管理" as a6
  usecase "系统设置(钉钉/报修选项)" as a7
}

t --> u1
t --> u2
t --> u3
t --> u4
t --> u5
g --> u6

a --> a1
a --> a2
a --> a3
a --> a4
a --> a5
a --> a6
a --> a7

@enduml
```

## 图4 报修工单状态流转图（活动/状态图）

```plantuml
@startuml
skinparam state {
  BackgroundColor #FDF6E8
  BorderColor #A4232A
}
title 报修工单状态流转

[*] --> 待处理 : 教师提交报修

待处理 --> 处理中 : 管理员/钉钉认领
待处理 --> 已取消 : 教师取消(仅待处理时)
待处理 --> 已解决 : 直接解决

处理中 --> 已解决 : 处理完成
处理中 --> 已关闭 : 关闭

已解决 --> 已关闭 : 归档
已解决 --> 处理中 : 重新打开(可选)

已取消 --> [*] : 终态(锁定)
已关闭 --> [*]

note right of 待处理
  可催单(2小时限一次)
  超时自动标红
end note
@enduml
```
